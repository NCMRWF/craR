Contiguent Rainfall Area (CRA) Analysis Using R

Raghavendra Ashrit, Anumeha Dube and Greeshma Mohan

08 May 2023

Contiguent Rainfall Area (CRA) is an entity based verifcation method developed by Ebert, E. E., and McBride, J. L. (2000). A CRA is defined for an observation-forecast pair based on user specified rainfall threshold and union of the forecast and observed rain entities. This method quantifies the systematic bias in the model in predicting the rainfall and dissociates it into different error components. The CRA analysis technique is well desribed in Ebert, E. E., and Gallus, W. A. (2009).

A brief summary of the CRA analysis procedure based on the minmization of the mean squared error (MSE) is given below,

  • Identification and pairing of the objects: The rainfall objects based on a defined threshold is identified from both forecast and observation using FeatureFinder from the SpatialVx package. The objects which are close by 5 gridcells are paired and objects which are smaller in area are ignored.
  • Calculation of the geometric distance between the centers of gravity (COG), mean square error and correlation between the identified objects.
  • The model field is translated over the CRA until a best match is obtained through the minimization of the MSE
  • The total error (MSETotal) decomposes into different components Ebert, E. E., and McBride, J. L. (2000).

\[ \tag{1} MSE_{Total} = MSE_{Displacement} + MSE_{Volume} + MSE_{Pattern} \]

The details on this package and the step-by-step instructions are described below.

1. Download CRA_package_V1.tar.gz and untar it.
It includes the following files

  • Isolate_Objects.R : Function to identify and pair rainfall objects from both model and observation based on rainfall threshold. This will provide each objects in separate NetCDF file.
  • CRA_Err_Decomp_Features.R : Main function for CRA analysis
  • Func_Run_CRA_Analysis.sh : This is a shell script which creates a R script which does CRA analysis for all the identified model-observation object pairs from the domain. Both of the aforementioned functions are used in this.
  • Run_CRA_Plot.sh : Creates figure including the necessary statistics tables
  • obs_20210517.nc : Test data for observation
  • model_20210517.nc : Test data for model forecast
  • output : This directory includes the output of this test case
  • ReadMe.txt :

3. Define the threshold rainfall
To identify the rainfall objects from model and observation threshold rainfall has to be defined in mm/day

Threshold<-40

4. The identification of rainfall objects
The Isolate_Objects.R identifies the rainfall objects and save them into NetCDF format. More details on the input arguments are given in Annexure.

source("Isolate_Objects.R")
Isolate_Objects("obs_20210517.nc","model_20210517.nc","Obj_Obs","Obj_Model",thr=40,minsize=30,sepdist=5)

The isolated objects are saved as Obj_Obs_Object1.nc and Obj_Model_Object1.nc from observation and model forecast respectively.
5. CRA Analysis
Using the aforementioned object files the errors are calculated using CRA_Err_Decomp function as follows. More details on the input arguments are given in Annexure.

source("CRA_Err_Decomp_Features.R")
CRA_Err_Decomp("obs_20210517.nc","model_20210517.nc","Obj_Obs_Object1.nc","Obj_Model_Object1.nc",thr=40,objnum=1,"20210517_Thr40_CTL",Ngrids=6)
The CRA_Err_Decomp_Features.R produces a set of ASCII files with a defined prefix (Here, 20210517_Thr40_CTL*.txt).
  • 20210517_Thr40_CTL_1_xydistcog.txt : Distance (in km) between the centre of gravity before and after shifting
  • 20210517_Thr40_CTL_1_cra_decomp.txt : CRA error components (%)
  • 20210517_Thr40_CTL_1_cog.txt : centre of gravity in terms of latitude and longitude before and after shifting the model
  • 20210517_Thr40_CTL_1_StatsOrig.txt : Stats (no. of points, average rainfall, maximum rainfall, rain volume, and area) from both observation and before shifting the model
  • 20210517_Thr40_CTL_1_ScatterData.txt : Data to plot scatter diagram
  • 20210517_Thr40_CTL_1_AllStats.txt : All stats (no. of points, average rainfall, maximum rainfall, rain volume, area, RMSE and correleation) from observation and after shifting the model field

6. Plotting the statistics
The final CRA plot with all the statistics before and after the shift of the model raster over the CRA area can be plotted using Run_CRA_Plot.sh.



7. CRA Analysis for multiple objects
One can use Func_Run_CRA_Analysis.sh, if multiple paired objects are to be analysed over the same domian. More details on this script can be found in ReadMe.txt.
